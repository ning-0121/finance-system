# 重复付款防控 SOP（制度，立即执行）

> 背景：曾发生货代费重复付款（已人工追回）。本制度用于杜绝再次发生，先于系统硬化生效。
> 适用：所有对外付款（货代费、加工费、面辅料款、税费等），尤其货代费。

## 一、一票一号，先核后付（录入端）
1. 每一张供应商账单/货代账单**必须登记单据号**：货代费用**报关单号或航次号**，其他用**发票号/账单号**。
2. 出纳付款前，先在系统按"供应商 + 单据号/金额"搜索近期付款，**确认这一票没付过**再付。
3. 同一报关单/航次的货代费**一票一付**，不拆不并，避免对不上。

## 二、货代费及大额付款双人确认（审批端）
4. 货代费，以及金额 ≥ ¥[由财务经理设定阈值] 的付款，**制单人与付款人必须分开**。
5. 付款前由**财务经理过目**确认无重复、单证齐全（账单 + 报关单 + 合同/订单对得上）。

## 三、每日异常巡查（检测端）
6. 财务经理每天上班**先看「控制中心 → 异常中心」的严重项**，重点看"疑似重复付款"。
7. 命中疑似重复 → 立即核实，确认无误后在异常中心填原因关闭；确实重复立即处理。

## 四、月度对账兜底（兜底端）
8. 每月向货代及主要供应商**索取对账单**，与系统付款流水逐笔核对。
9. 启用「银行对账」：导入银行流水，确保每一笔支出都能对上唯一一笔应付——重复付款在银行流水层必现形。

## 责任分工
- **录单（财务员）**：登记单据号、付款申请。
- **审核（财务经理）**：付款前核重、看每日异常、月度对账。
- **付款（出纳）**：付款前先搜重，按审批执行。
- **复核（财务负责人/老板）**：收严重异常企微告警，月度抽查。

---
本 SOP 即日生效，系统硬化（单据号唯一约束 / 付款前实时查重 / 职责分离 / 实时告警）上线后，部分环节将由系统自动拦截，但人工核对制度持续有效。
