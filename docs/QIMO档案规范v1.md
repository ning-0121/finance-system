# QIMO 档案规范 v1（微盘统一归档）

> 适用范围：绮陌 OS 全线（业务开发 / 订单节拍器 / 财务系统）。
> 本规范是唯一路径与命名标准，两侧仓库各存一份，改版需双方确认并升版本号。

## 0. 架构铁律

1. **一份目录树**：所有系统的文件落同一棵树（根：`QIMO档案/`），禁止任何系统另起根目录。
2. **目录单一管家**：微盘文件夹允许同名、无路径唯一性 → **只有财务系统（归档中枢）可创建文件夹**，
   并在注册表 `fin_drive_nodes`（路径字符串唯一 → 微盘 fileid）登记。
   其他系统通过财务侧「路径解析接口」换取文件夹 ID，**只传文件、不建夹**。
3. **上传职务分离**：各系统用**自己的企微应用身份**上传（审计归属清晰）；
   每个单据编码段只属于一个系统（见 §2），文件级不重复。
4. **共享 ID 推导路径**：路径只由「内部单号 / 客户名 / 供应商名 / 工厂名 / 年月」推导，
   这些 ID 两系统经 webhook 已同步，任何一侧算出的路径字符串必须一致。
   客户/供应商名先做归一化（NFKC、去首尾空格、全角转半角）再入路径。

## 1. 目录树

```
QIMO档案/
├── 01-订单线/{YYYY}/{客户}/{内部单号}/      ← 一单一夹（业务+订单+财务共用）
├── 02-客户线/{客户}/                        ← 跨单客户资料（对账单/收款凭证/信保）
├── 03-供应商线/{供应商}/{YYYY}/             ← 合同/发票/对账单
├── 04-工厂线/{工厂}/{YYYY}/                 ← 加工合同/结算单/出货记录
└── 05-费用线/{YYYY}/{MM}/                   ← 行政/工资/其他
```

- `{YYYY}` 取单据业务日期年份（订单线取下单年）；`{MM}` 两位月份。
- 一单一夹内**不再建子文件夹**，靠文件名编码区分单据类型（便于整夹打包与检索）。

## 3. 文件命名

```
{内部单号或对象名}_{编码}{单据类型}_{YYYYMMDD}_{对方简称}[_v{n}].{ext}
```

示例：
- `510B_12CI_20260618_rag.pdf`
- `510B_06采购单_20260415_华航布行.pdf`
- `华航布行_33供应商对账单_20260630.xlsx`
- 同名重传自动追加 `_v2/_v3`（由归档中枢处理，不覆盖旧版）。

日期 = 单据业务日期（非上传日期）。检索优先靠文件名（跨线搜索用微盘全局搜索）。

## 4. 接口契约（订单侧对接）

1. **路径解析**：`POST {财务系统}/api/integration/drive/resolve`
   鉴权：同 webhook（x-api-key + HMAC 签名，对 raw body）。
   请求：`{ line: '01'|'02'|'03'|'04'|'05', keys: { year, customer?, internal_order_no?, supplier?, factory?, month? } }`
   响应：`{ spaceid, fatherid, path }`（不存在则由财务中枢创建后返回；注册表保证幂等）。
2. **上传**：订单侧拿 `spaceid/fatherid` 用**自己的应用身份**调微盘 `file_upload`。
3. **回执（可选）**：上传成功后按既有 webhook 发 `file.uploaded`（带 fileid/编码/单号），财务侧登记档案台账。
